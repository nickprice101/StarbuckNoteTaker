STARBUCK NOTE TAKER - QWEN DEPLOYMENT
=====================================

ACTIVE AI RUNTIME
-----------------

- Model: Qwen3 0.6B mixed-int4
- Artifact: qwen3_0_6b_mixed_int4.litertlm
- Runtime: Google LiteRT-LM for Android
- Download: managed by LlamaModelManager on first use (~475 MB)
- Supported ABIs: arm64-v8a and x86_64
- Minimum device RAM: 4 GB

PROMPTS
-------

Edit config/AI_AGENT_PROMPTS.txt at the repository root. It must contain:

- [AI_SUMMARISER]
- [AI_CHATBOT]
- [AI_REFORMATTING]

The Gradle preBuild pipeline copies the file into generated APK assets. AiAgentPrompts validates
and loads the sections, and LlamaEngine routes summary, rewrite, and question modes through Qwen.

BUILD
-----

Requirements:

- JDK 17
- Android SDK platform 34
- Android build tools 34.0.0
- Android NDK 26.1.10909125

Commands:

  ./gradlew test --no-daemon --console=plain
  ./gradlew assembleDebug --no-daemon --console=plain

No Python AI package, mlc-llm wheel, TVM runtime compilation, TensorFlow Lite classifier, category
mapping, or classifier vocabulary is required.

RUNTIME CHECKS
--------------

1. Open Settings -> AI model.
2. Download the Qwen model and allow checksum validation to complete.
3. Verify summary, rewrite, and question modes on a supported device.
4. Confirm the app uses GPU then CPU on ARM64, or CPU on x86_64.
5. Confirm model-unavailable behavior produces only the documented plain fallback UI.
