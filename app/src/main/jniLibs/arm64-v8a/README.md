# `jniLibs/arm64-v8a/` — native libraries

This directory is the standard Android location for pre-built native shared
libraries (`.so` files).  The Gradle build packages everything here into the APK
automatically.

## MLC LLM model library — modern `.tar` flow

The project now uses the **modern `.tar` flow** introduced in MLC LLM.  Instead of
requiring a separately compiled `.so` file here, the model library is distributed
as a gzip-compressed tar archive:

```
app/src/main/assets/Llama-3.2-3B-Instruct-q4f16_0-MLC-android.tar
```

The archive contains the compiled MLC system-library object files:
- `lib0.o` — TVM runtime support code
- `llama_q4f16_0_devc.o` — quantised Llama device code

On the first inference attempt `LlamaModelManager.extractModelLibIfNeeded()` extracts
the archive to `filesDir/lib/Llama-3.2-3B-Instruct-q4f16_0-MLC-android/` and the
resulting path is passed to `MLCEngine.reload()` as `modelLib`.

No manual compilation step is required — the `.tar` is already committed as an APK
asset and is extracted automatically at runtime.

## Other `.so` files

Any other native libraries (e.g. `libdjl_tokenizer.so`, `libc++_shared.so`) are
still generated by the Gradle build and placed here automatically.  You do not need
to commit them manually.
